# Appendices

## Appendix A: Common GraphQL Anti-Patterns
- Over-fetching/under-fetching due to poor schema boundaries.
- N+1 resolver patterns without batching/data loaders.
- Leaky internal errors and inconsistent error contracts.
- Unbounded query complexity/depth and missing governance controls.

## Appendix B: Glossary of Terms
- Type system concepts: object, interface, union, scalar, enum, input.
- Runtime concepts: resolver, execution context, introspection, fragments.
- Scale concepts: federation, persisted queries, schema registry.
